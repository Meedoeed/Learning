# CiscoPT



## Схема: Два свитча 2960. 


    Свитч A: PC0 в VLAN 10, PC1 в VLAN 20

    Свитч B: PC2 в VLAN 10, PC3 в VLAN 20

    Между свитчами — один кабель
Итог: PC0 пингует PC2, но не пингует PC3, так как он в другом влане

![topology](images/daily0204.png)
![terminal](images/daily0204(2).png)
### Вывод
1) При соединении двух коммутаторов access-портом сеть падает при отсутствии STP, а если STP есть - один из портов коммутаторов блокируется (аварийное состояние)
2) Для взаимодействия двух коммутаторов нужен trunk-порт
Access - конечные устройства (отбрасывает VLAN-тег)
Trunk - коммутаторы и маршрутизаторы
3) В отличие от обычного Ethernet-кадра по транку передаётся и служебная информация в виде vlan-тега
4) Если влан тега нет - кадр уходит на VLAN 1 (Native vlan - по умолчанию)

### Полезно
- show interfaces trunk
- show interfaces fastEthernet 0/1 switchport



## Схема: Свитч 2960 + роутер 1941 + два PC в разных VLAN.

Задание:
Сделай так, чтобы PC0 (VLAN 10) и PC1 (VLAN 20) пинговали друг друга через роутер.

Итог - на роутере было сделано 2 сабинтерфейса и PC4 и 5 пингуются через роутер
![topology](images/daily0204(3).png)
![terminal](images/daily0204(4).png)

### Вывод
1) Роутер оперирует IP-пакетами и не видит VLAN-теги. Нужно научить его распознавать их при приёме с коммутатора.
VLAN - L2
Router - L3
Роутер смотрит на тег только в случае настроенного сабинтерфейса с encapsulation dot1Q. Без него роутер просто отбросит тег
2) Сабинтерфейс - виртуальный интерфейс на физ. порту роутера. Он позволяет обслуживать несколько VLAN через один кабель
Создание сабинтерфейсов
``` cisco 
interface gigabitEthernet 0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
 no shutdown
 exit
 ```
 3) Маршрутизация на вланы включается автоматически при настроенных сабинтерфейсах с IP разных подсетей.

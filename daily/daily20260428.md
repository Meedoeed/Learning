## Iptables

### Задание 1

Твоя цель — настроить "белый список" для доступа к управлению сервером.

    Узнай IP-адрес своей хостовой машины (той, с которой ты подключаешься к VM) или выбери любой произвольный IP в твоей сети, который ты назначишь «доверенным».

    Разрешите этому конкретному IP подключаться к порту 22 (SSH).

    Разрешите трафик на интерфейсе lo (loopback). 

    Установите политику по умолчанию для цепочки INPUT в состояние DROP.

### Решение 1

sudo iptables -A INPUT -p tcp --dport 22 -s 192.168.1.132 -j ACCEPT
sudo iptables -A INPUT -i lo -j ACCEPT
sudo iptables -P INPUT DROP

### Задание 2 

Твоя задача — сделать так, чтобы сервер мог полноценно выходить в интернет, оставаясь закрытым для новых входящих соединений (кроме твоего SSH из первого задания).

    Убедись, что сейчас твой сервер "молчит": попробуй выполнить ping 8.8.8.8 или curl google.com. Если политика INPUT стоит в DROP, ответы приходить не будут.

    Добавь правило в цепочку INPUT, которое разрешает пакеты в состояниях ESTABLISHED и RELATED.

    Проверь результат: попробуй снова ping 8.8.8.8. Теперь ответы должны проходить.

    Посмотри список правил с флагом -v. Обрати внимание, как растут счетчики байтов напротив нового правила, когда ты что-то скачиваешь или пингуешь.

### Решение 2 

sudo iptables -A INPUT -m conntrack -ctstate ESTABLISHED,RELATED -j ACCEPT

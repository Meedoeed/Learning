## TCP/IP
Ознакомился с уровнями стека TCP/IP
- прикладной: SMTP, FTP, HTTP, telnet
- транзитный: TCP, UDP
- сетевой: IP, ICMP, RIP 
- уровень сетевых интерфейсов

Понял для чего предназначен каждый урвоень, в чём различия между уровнями этого стека и стека OSI. 

Для TCP
Прикладной уровень -поток-> UDP -дейтаграмма-> IP -пакет-> сетевые интерфейсы -кадры->
Для UDP
Прикладной уровень -сегмент-> UDP -дейтаграмма-> IP -пакет-> сетевые интерфейсы -кадры->

В дальнейшем буду ознакамливаться с каждым протоколом подробнее в отдельности

## Практика 
Началл выполнять лабораторную работу

Настроить минимальную безопасность доступа на двух коммутаторах и убедиться, что аутентификация работает, а неавторизованный доступ блокируется.
[PC] — управляющий хост (192.168.1.10)
         |
    [Switch1] — порт управления (vlan 1: 192.168.1.1)
         |
    [Switch2] — (192.168.1.2)

Задание:
Часть А. Базовая защита на Switch1

    Установите hostname и domain-name (требуется для SSH).

    Сгенерируйте RSA-ключ (длина 2048 бит).

    Создайте локального пользователя с секретным паролем (не в открытом виде).

    Настройте аутентификацию на VTY-линиях:

        Только SSH (запретите telnet)

        Использовать локальную БД пользователей

        Установите таймаут сессии 5 минут

        Разрешите доступ только с одного IP-адреса (вашего PC)

    Настройте пароль на привилегированный режим (enable secret) — тип хэша 8 или 9.

    Настройте пароль на консоль, но оставьте локального пользователя как приоритетный.

    Добавьте баннер с предупреждением.

Часть Б. Работа с уровнями на Switch2

    На Switch2 настройте минимальную защиту:

        Enable secret (другой пароль)

        Один пользователь с доступом только к show командам (уровень 1 или 15? Подумайте, как ограничить)

    Настройте возможность удаленного управления с Switch1 (SSH).

        Разрешите VTY-доступ только с IP Switch1.
